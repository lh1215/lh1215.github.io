1.对象劫持

我们先来看看vuejs对象劫持的流程：

_init() => initState() => initData() => observe() => new Observe() => this.walk() => defineReactActive()

下面我们一步一步看，这些流程方法究竟做了些什么。

  • _init()

(⊙o⊙)…，其实这里啥也没有干，就是初始化调用了一个方法，把数据传给了_init()方法。

  • initState()

这里主要是对一系列方法的初始化，比如data，props，watch等等。。。。。。,我们这里只分析initData()方法

  • initData()

这里做的事情其实也很简单，主要是对data的类型判断和处理，vuejs里data方法传的可能是一个function或者一个对象，如果传的是一个对象那么直接使用，但是如果是返回一个function，这个时候需要使用call方法改变this指向，并且执行这个方法得到对象。

  • observe(data)

这里面做的操作其实就是判断一下data是否是对象，可能有童鞋会有疑问，为什么这里需要判断是否是对象呢？其实这里是为了后需的递归才加了判断的，后面揭晓。

  • new Observe(data) => this.walk()

这里包括了一个很重要的步骤，Object.kyes把对象所有的key拿出来并且组成数组，让forEach去循环，并且把元对象：data，字段名：key以及对应的对象value值：data[key]传给defineReactActive。相信大家都猜到接下来要干嘛了，对象，对象key，value值都齐全了，接下来要干嘛了？没错，那就是使用Object.defineProperty进行数据劫持了。

  • defineReactActive(data, key, value)

ok，终于到重头戏了。Object.defineProperty就是vuejs进行双向绑定的核心了，这里会检测对象中的每一个值，如果值重新赋值了，它可以监听到值的变化。
可能眼尖的童鞋已经发现了这里还调用了两个observe(value)、observe(newValue),那么这里又是做了什么呢？这里其实就是上面我说到的递归了，为什么需要这两个递归？

第一个observe()，这里是因为可能传过来的对象是多层嵌套的，就是对象里面还有对象，这个时候我也需要对对象里面的对象再次进行监测，所以这里加了一个observe方法。

第二个observe()，这里是因为可能在重新给值赋值的时候赋的是一个对象，这个时候也是需要重新监测的。
